# סיכום פתרונות טכניים - אופטימיזציה של Gemini Live ו-Firestore

מסמך זה מרכז את הבעיות הטכניות שנתקלו בהן במהלך פיתוח הצ'אט הקולי והטקסטואלי של ג'ורג' מקינזי, והפתרונות המדויקים שיושמו.

---

## 1. גישה ל-Gemini API (צד לקוח מול צד שרת)
**הבעיה:** שגיאת `API_KEY_SERVICE_BLOCKED` או חסימת `AuthTokenService.CreateToken` כאשר מנסים לייצר מפתחות זמניים (Ephemeral Tokens) דרך השרת.
**הפתרון:** 
* העברת כל הלוגיקה של Gemini (טקסט וקול) ישירות ל-Frontend.
* הזרקת המפתח ב-`vite.config.ts` באמצעות `define`:
  ```typescript
  define: {
    'process.env.GEMINI_API_KEY': JSON.stringify(process.env.GEMINI_API_KEY || "")
  }
  ```
* שימוש ישיר ב-`GoogleGenAI` בדפדפן.

## 2. אופטימיזציה של השהיה (Latency) בשיחה קולית
**הבעיה:** תגובות איטיות מדי של ה-AI בשיחה קולית.
**הפתרון:**
* **הקטנת Buffer:** שינוי גודל ה-Buffer ב-`createScriptProcessor` מ-4096 ל-1024. זה מקטין את זמן עיבוד האודיו לפני שליחתו ל-AI.
* **הנחיית תמציתיות:** הוספת הוראה ב-System Prompt ל-AI להיות "תמציתי באופן קיצוני" (1-2 משפטים). ככל שהתשובה קצרה יותר, ה-Streaming מתחיל מהר יותר.

## 3. יציבות שמע ומניעת קיטועים
**הבעיה:** הקול מתחיל מהאמצע, נשמע קטוע או סובל מ"קליפינג" (Clipping) דיגיטלי.
**הפתרון:**
* **השהיית חיבור:** הגדלת ה-`setTimeout` לפני שליחת משפט הפתיחה ל-800ms כדי לוודא שה-WebSocket יציב לחלוטין.
* **Gain Node:** הוספת `GainNode` עם ערך של 0.95 בשרשרת השמע כדי למנוע עיוותים ועוצמת יתר שגורמת לקיטועים.

## 4. ניהול משפט פתיחה וזהות ה-AI
**הבעיה:** ה-AI מתבלבל בזהויות (קורא לעצמו בשם המשתמש) או ממשיך לדבר ללא הפסקה בתחילת השיחה.
**הפתרון:**
* **הוראה ישירה בחיבור:** שליחת `sendRealtimeInput` מיד עם פתיחת ה-Session עם הוראה מפורשת: "תגיד בדיוק [משפט] ואז תשתוק ותקשיב".
* **הפרדת תפקידים ב-Prompt:** הגדרה ברורה ב-System Prompt: "זכור שאתה ג'ורג' (האסטרטג) והמשתמש הוא הלקוח".

## 5. מנגנון קטיעה (Interruption / Barge-in)
**הבעיה:** ה-AI לא מפסיק לדבר כשהמשתמש מתפרץ לדבריו.
**הפתרון:**
* **זיהוי RMS:** חישוב עוצמת הקול (RMS) של המשתמש ב-Audio Processor.
* **הורדת סף רגישות:** הגדרת סף של 0.012. ברגע שהמשתמש מדבר מעל עוצמה זו, מופעלת פונקציית `stopPlayback()` שמפסיקה מיידית את כל מקורות השמע הפעילים של ה-AI.

## 6. הרשאות Firestore למשתמשים אנונימיים
**הבעיה:** שגיאת `Missing or insufficient permissions` בעדכון מונה הודעות (Rate Limiting) עבור מבקרים שאינם מחוברים.
**הפתרון:**
* פישוט חוקי האבטחה ב-`firestore.rules` כך שיאפשרו `get`, `create` ו-`update` על סמך מזהה ייחודי (למשל מ-LocalStorage) עם וולידציה על תוכן השדה (למשל `count <= 100`).
